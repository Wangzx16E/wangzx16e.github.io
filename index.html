MongoDB在Windows上安装和连接VS2012.

发表于 2017-03-27

MongoDB在Windows上安装.

注明：我根据网络博文安装运行成功，这是超简单版。其他快捷启动，查询等讲解请移步Lecaf。

1.官网根据版本下载。

2.解压缩到自己想要安装的目录，比如d:\Mongodb

3.创建文件夹d:\Mongodb\db 用来安装db

4.运行cmd.exe进入dos命令界面，执行下列命令

>cd d:\Mongodb\Server\bin   
>mongod -dbpath "d:\Mongodb\db"

5.测试连接：新开一个cmd窗口，转目录到bin所在的文件夹

>cd d:\Mongodb\Server\bin   
>mongo

6.这样就可以运行了：）麻烦就在于每次运行都要cd再输入2次，可以装一个MongoDB Compass（GUI），或者移步Lecaf设置日志快捷登录。

7.关于MongoDB Compass，有最基本的分析同一个Field的数据分布成图功能，可以去官网瞅一眼。

MongoDB连VS2012

MongoDB官网上提供的NuGet安装方式，对版本要求太高，我的VS2012上安装不了NuGet2.12版本，于是所有的都无法安装。我用的是VS2012中工具菜单里扩展和更新，选择联机， 搜索Mongo找到的 CData.ADO.NET.MongoDB provider (官网)。

关于NuGet

需要说明的一点是，在NuGet的管理程序包中容易出现

无法连接远程服务器。

我查了挺多资料，一是说需要自己设proxy，并且在config文件中手改一些数据。我试了几次没成；二是说问题是nuget.org在国内不易连上，我尝试用翻墙VPN，依旧显示无法连接。
最后，我去官网的package列表里手动下载的包，放在一个文件夹里，把文件夹目录手动添加到nuget程序包源里，就可以在VS里看到这些package了。(写文章的时候又去了Github查了一下，mongo-csharp-driver)

注: 可以直接下载的连接是在package页面的左侧download(how to)，所以只能下载官网里有的包版本。如果需要早期版本:( ,可以在GitHub上搜索看年份选取。

MongoDB导入shp并使用Geospatial Index

发表于 2017-03-20

MongoDB导入shp文件

1.将shp转为geoJSON，使用GDAL的命令行工具ogr2ogr。

ogr2ogr -f geoJSON place.json place.shp

2.删掉开头2行和最后1行，变成jsonArray。
3.用mongoimport进行导入。 

mongoimport --db <db-name> --collection <coll-name> --type json --file place.json--jsonArray

Geospatial Index

1.数据如下：
库：users
collection： place

MongoDB compass

2.添加Geospatial Index

终端

使用createIndex()添加 Geospatial Index

db.place.createIndex({ geometry: "2dsphere"})

3.使用$geoWithin查询Geospatial Index

终端
